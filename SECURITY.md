# Security policy

## Reporting a vulnerability

Do not open a public GitHub issue for security problems.

### Preferred channel (when available)

When private vulnerability reporting is enabled on this repository, report here:

https://github.com/kaybi-gh/K7/security/advisories/new

That form is private to maintainers and the reporter. Enable it under **Settings -> Code security -> Private vulnerability reporting** (typically once the repository is public).

### While private vulnerability reporting is unavailable

On a private repository, or when private vulnerability reporting is disabled, `/security/advisories/new` returns 404.

Contact the maintainer privately instead:

- GitHub: [@kaybi-gh](https://github.com/kaybi-gh)

Include steps to reproduce, affected versions, and impact. Do not disclose the issue publicly until a fix is available or you are told it is safe to do so.

## Supported versions

Security fixes are applied to the latest release when possible. Older tags may not receive backports.
