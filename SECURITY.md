# Security policy

## Reporting a vulnerability

Do not open a public GitHub issue or discussion for security problems.

Contact the maintainer privately:

- GitHub: [@kaybi-gh](https://github.com/kaybi-gh)

Include steps to reproduce, affected versions, and impact. Do not disclose the issue publicly until a fix is available or you are told it is safe to do so.

### Private vulnerability reporting (optional)

After this repository is public, you can enable **Settings -> Code security -> Private vulnerability reporting**. The private form then appears at:

`https://github.com/kaybi-gh/K7/security/advisories/new`

Until that setting is on, that URL returns 404. Use the maintainer contact above.

## Supported versions

Security fixes are applied to the latest release when possible. Older tags may not receive backports.
