# Impeccable Design Context

## Design Context

### Users
Self-hosters and their family and friends - a small, trusted circle that shares a private media server. No cloud accounts, no subscriptions: everything is local and personal. Usage spans all three contexts: desktop (focused, seated), mobile (on the go, one-handed), and couch/TV (10-foot UI, remote or touch). The same person may use K7 in all three contexts in the same day.

### Brand Personality
**Cinematic. Yours. Understated.**

K7 is your personal cinema - intimate, private, artisanal. It has the polish of a premium streaming service but none of the corporate coldness. It doesn't shout; it shows. The brand should feel like a curated screening room, not a SaaS dashboard.

Anti-reference: neon aesthetics, glowing cyan/purple accents, anything that competes visually with the content itself.

### Aesthetic Direction
- **Dark-first, cinema-coded**: Deep blue-gray backgrounds (`#0c1018`-`#131821`) are the default canvas. Dark mode is the primary experience - it matches how people actually watch media.
- **Content is the hero**: Artwork (posters, covers, stills, album art) carries the visual weight. The UI chrome should recede and frame, never compete. Let images breathe.
- **Extracted color, not imposed color**: Where colour is needed for atmosphere (album detail hero blur, fullscreen player background), it should be pulled from the content itself - not from a fixed palette.
- **Light mode is a first-class citizen**: When light mode is active, it should feel equally refined - cool off-whites, soft surfaces, not a blindingly white inversion.
- **Theme-ready token architecture**: The copper primary (`#CC7A3E`) defines the brand accent. All colour decisions must use K7 design tokens (CSS variables) - never hard-coded hex values - so users can eventually create and install custom themes.
- **Typography communicates hierarchy**: Epilogue (headings) for cinematic weight and presence; Manrope (body) for clarity and readability at small sizes. No monospace as a "technical" aesthetic shortcut.

### Design Principles

1. **Content first, chrome last** - Every UI element should ask "does this help the user get to or enjoy their media?" If not, remove or reduce it. Artwork, posters, and covers are the primary visual language.

2. **Cinematic restraint** - Use darkness and space deliberately. Avoid neon accents, glow effects, or decorative gradients. When color appears, it should feel earned and contextual (extracted from content, not imposed by the brand).

3. **Every context is a first-class citizen** - Desktop, mobile, and 10-foot couch mode are each real use cases. Do not shrink the desktop layout for mobile - adapt the interaction model. Do not ignore couch mode.

4. **Tokens over hard-codes, always** - The theme system is a product feature. Any color, spacing, or typography value that matters must reference a design token so users can eventually swap themes without touching code.

5. **Accessible by default** - WCAG AA is the floor, not the ceiling. Sufficient contrast, keyboard navigation, meaningful focus indicators, and semantic markup benefit all users - including those in low-light couch environments or on small mobile screens.
